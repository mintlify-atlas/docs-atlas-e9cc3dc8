title: Timeouts and Retries
description: Configure timeout and retry behavior for robust API interactions
---

The Stagehand SDK provides flexible timeout and retry configuration to handle network issues, slow operations, and transient failures gracefully.

## Default Behavior

By default:
- **Timeout**: 60 seconds (1 minute)
- **Max Retries**: 2 retries with exponential backoff
- **Retried Errors**: Connection errors, 408, 409, 429, and 5xx status codes

## Configuring Timeouts

### Client-Level Timeout

Set a default timeout for all requests:

```python
from stagehand import AsyncStagehand

# Set 30 second timeout for all requests
client = AsyncStagehand(timeout=30.0)

# Disable timeout entirely (not recommended)
client = AsyncStagehand(timeout=None)
```

### Request-Level Timeout

Override the timeout for specific operations:

```python
async with AsyncStagehand() as client:
    session = await client.sessions.start(
        model_name="anthropic/claude-sonnet-4-6"
    )
    
    # Fast operation - short timeout
    await session.act(
        input="click button",
        timeout=10.0  # 10 seconds
    )
    
    # Long operation - extended timeout
    await session.execute(
        execute_options={
            "instruction": "Complete complex multi-step task",
            "max_steps": 20
        },
        agent_config={"model": "anthropic/claude-opus-4-6"},
        timeout=300.0  # 5 minutes
    )
```

### Advanced Timeout Configuration

Use `httpx.Timeout` for fine-grained control:

```python
import httpx
from stagehand import AsyncStagehand

# Different timeouts for different phases
timeout = httpx.Timeout(
    connect=5.0,   # 5s to establish connection
    read=60.0,     # 60s to read response
    write=10.0,    # 10s to send request
    pool=5.0       # 5s to acquire connection from pool
)

client = AsyncStagehand(timeout=timeout)
```

## Handling Timeout Errors

```python
import asyncio
import stagehand

async def handle_timeouts():
    async with AsyncStagehand() as client:
        session = await client.sessions.start(
            model_name="anthropic/claude-sonnet-4-6"
        )
        
        try:
            result = await session.execute(
                execute_options={
                    "instruction": "Complete task",
                    "max_steps": 10
                },
                agent_config={"model": "anthropic/claude-opus-4-6"},
                timeout=60.0
            )
        except stagehand.APITimeoutError:
            print("Operation timed out after 60 seconds")
            
            # Retry with longer timeout
            print("Retrying with extended timeout...")
            result = await session.execute(
                execute_options={
                    "instruction": "Complete task",
                    "max_steps": 10
                },
                agent_config={"model": "anthropic/claude-opus-4-6"},
                timeout=180.0  # Try 3 minutes
            )
```

## Configuring Retries

### Client-Level Retries

```python
from stagehand import AsyncStagehand

# No retries
client = AsyncStagehand(max_retries=0)

# Increase retries for unreliable networks
client = AsyncStagehand(max_retries=5)

# Default is 2 retries
client = AsyncStagehand()  # max_retries=2
```

### Request-Level Retries

```python
async with AsyncStagehand() as client:
    session = await client.sessions.start(
        model_name="anthropic/claude-sonnet-4-6"
    )
    
    # Disable retries for this request
    result = await client.with_options(max_retries=0).sessions.act(
        session.id,
        input="click button"
    )
    
    # Increase retries for critical operation
    result = await client.with_options(max_retries=5).sessions.act(
        session.id,
        input="submit important form"
    )
```

## What Gets Retried

The SDK automatically retries these errors:

- **Connection Errors**: Network connectivity issues
- **408 Request Timeout**: Server-side timeout
- **409 Conflict**: Temporary resource conflicts
- **429 Rate Limit**: Too many requests
- **500+ Internal Server Errors**: Server-side failures

```python
try:
    result = await session.act(input="click button")
except stagehand.RateLimitError:
    # This was already retried 2 times with backoff
    print("Still rate limited after retries")
```

## Exponential Backoff

Retries use exponential backoff automatically:

- **1st retry**: Wait ~1 second
- **2nd retry**: Wait ~2 seconds
- **3rd retry**: Wait ~4 seconds
- And so on...

```python
import time
import stagehand

start = time.time()

try:
    # This will retry on transient failures
    result = await session.act(
        input="click button",
        max_retries=3
    )
except stagehand.APIError as e:
    duration = time.time() - start
    print(f"Failed after {duration:.1f}s and multiple retries")
```

## Custom Retry Logic

Implement custom retry strategies:

```python
import asyncio
import random
from typing import TypeVar, Callable
import stagehand

T = TypeVar('T')

async def custom_retry(
    func: Callable[[], T],
    max_attempts: int = 3,
    base_delay: float = 1.0,
    max_delay: float = 60.0,
    exponential_base: float = 2.0,
    jitter: bool = True
) -> T:
    """Custom retry with exponential backoff and jitter."""
    
    for attempt in range(max_attempts):
        try:
            return await func()
        except (
            stagehand.APIConnectionError,
            stagehand.APITimeoutError,
            stagehand.RateLimitError,
            stagehand.InternalServerError
        ) as e:
            if attempt == max_attempts - 1:
                # Last attempt failed
                raise
            
            # Calculate delay
            delay = min(
                base_delay * (exponential_base ** attempt),
                max_delay
            )
            
            # Add jitter to prevent thundering herd
            if jitter:
                delay = delay * (0.5 + random.random())
            
            print(f"Attempt {attempt + 1} failed: {e.message}")
            print(f"Retrying in {delay:.2f}s...")
            await asyncio.sleep(delay)
    
    raise RuntimeError("Max retries exceeded")

# Usage
result = await custom_retry(
    lambda: session.act(input="click button"),
    max_attempts=5,
    base_delay=2.0,
    jitter=True
)
```

## Combining Timeouts and Retries

```python
async def robust_operation():
    """Operation with timeout and retry logic."""
    async with AsyncStagehand(
        max_retries=3,
        timeout=60.0
    ) as client:
        session = await client.sessions.start(
            model_name="anthropic/claude-sonnet-4-6"
        )
        
        try:
            # This will:
            # 1. Timeout after 60 seconds
            # 2. Retry up to 3 times on transient failures
            # 3. Use exponential backoff between retries
            result = await session.act(
                input="click button and submit form"
            )
            return result
            
        except stagehand.APITimeoutError:
            print("Timed out even with retries")
            # Try with longer timeout
            result = await client.with_options(
                timeout=120.0
            ).sessions.act(
                session.id,
                input="click button and submit form"
            )
            return result
```

## Streaming with Timeouts

Timeouts work with streaming responses:

```python
import asyncio

async def stream_with_timeout():
    async with AsyncStagehand() as client:
        session = await client.sessions.start(
            model_name="anthropic/claude-sonnet-4-6"
        )
        
        try:
            # Set timeout for the entire streaming operation
            async with asyncio.timeout(120.0):  # 2 minute total timeout
                stream = await session.execute(
                    execute_options={
                        "instruction": "Complete task",
                        "max_steps": 10
                    },
                    agent_config={"model": "anthropic/claude-opus-4-6"},
                    stream_response=True,
                    x_stream_response="true"
                )
                
                async for event in stream:
                    if event.type == "log":
                        print(event.data.message)
                    elif event.data.status == "finished":
                        return event.data.result
        
        except asyncio.TimeoutError:
            print("Streaming operation exceeded 2 minute timeout")
            return None
```

## Timeout for Action-Level Operations

Set timeouts for individual actions within operations:

```python
# Timeout for the act() API call itself
await session.act(
    input="click button",
    timeout=30.0  # Request timeout
)

# Timeout for the browser action execution
await session.act(
    input="click button",
    options={
        "timeout": 5000  # 5 seconds in milliseconds for the action
    },
    timeout=30.0  # Request timeout
)
```

## Best Practices

<Tip>
Set generous timeouts for agent execution. Complex tasks can take several minutes. Start with 300 seconds (5 minutes) or more.
</Tip>

<Warning>
Disabling retries (`max_retries=0`) can make your application fragile. Only do this for idempotent operations or during testing.
</Warning>

<Info>
Log timeout and retry information to understand operation performance and identify problematic patterns.
</Info>
